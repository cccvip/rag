**智能运维 Agent — LLM 驱动的故障自动诊断与自愈**

这是我把"任务调度与执行"从确定性流程推进到非确定性推理的一次实践。传统运维靠人工排查，我用 Agent 实现了"感知→推理→行动→闭环"的自动运维链路。

- **工具编排**：整合 ELK 日志聚合、Prometheus 指标监控、告警收敛为 Agent 工具集，通过 Function Calling 实现工具的动态调用与结果回传。
- **推理策略**：基于 LLM 实现故障根因分析，针对 MQ 堆积、DB 慢查询、Redis 热点 Key 等场景设计结构化 Prompt 模板，引导模型输出可执行的诊断结论。
- **行动执行**：Agent 根据诊断结果自动触发自愈动作（服务重启、流量切换、资源扩容），同时保留人工介入兜底。
- **效果闭环**：建立 Bad Case 归集与评测机制，持续优化 Prompt 与工具调用策略。
- **成果**：夜间告警人工介入率从 100% 降至 15%，MTTR 从 45 分钟压缩至 8 分钟。

---

**自研流程引擎中台 — 多租户任务调度与执行平台**

这是我在"确定性任务调度"方向上的集大成之作。从电商的 HTTP 请求调度，到流程引擎的业务节点调度，核心问题没变：**怎么高效、公平、可靠地把任务分发给执行单元，并在异常时快速失败。**

- **分片调度**：参考 Netty EventLoop 设计分片调度器，按 instance_id 哈希分片，多分片并行，天然保证实例级串行化，无需分布式锁，单机支撑 5000+ TPS。
- **节点执行**：业务方通过 SDK 编写 Handler，引擎自动完成任务派发、回调接收、超时检测。这个 SDK Handler 机制本质上就是 Agent 场景下的工具注册与执行。
- **状态持久化**：主键更新同步落盘 + 乐观锁，节点超时告警 + 人工介入，超时率 < 0.1%。这套"快速失败 + 人工介入"的设计原则，同样适用于 Agent 长周期任务的可靠性保障。
- **五层租户隔离**：入口限流 → 分片调度 → 分层执行 → 实例配额 → 租户熔断。大客户独立 Lane 精细化管控，中小客户按套餐分组共享，冷租户 LRU 自动淘汰。该方案可直接复用于 Agent 平台的租户级模型配置隔离与 Token 配额管控。
- **全链路可观测**：Prometheus + Grafana 覆盖各租户队列深度、Semaphore 占用、错误率 TopK、Redis 限流 RTT（P99 < 2ms）。该体系可迁移为 Agent 全链路追踪方案。
- **成果**：支撑 200+ 租户、TPS 5000+，单租户流量洪峰 10 倍时其他租户无感知，新产品 MVP 交付周期从 6 周压缩至 2 周，助力签约 2 家世界 500 强客户。
