**智能运维 Agent — LLM 驱动的故障自动诊断与自愈**

为保证平台 7×24 高可用，设计并落地基于 LLM 的智能运维 Agent，实现"感知→推理→行动→闭环"的自动运维链路，替代原人工 24 小时 On Call 模式。

- **工具编排**：整合 ELK 日志聚合、Prometheus 指标监控、告警收敛为 Agent 工具集，通过 Function Calling 实现工具的动态调用与结果回传。
- **推理策略**：基于本地部署的 Qwen 模型实现故障根因分析，针对 MQ 堆积、DB 慢查询、Redis 热点 Key 等场景设计结构化 Prompt 模板，引导模型输出可执行的诊断结论。本地化部署保证数据不出域，满足 TOB 客户的安全合规要求。
- **行动执行**：Agent 根据诊断结果自动触发预定义的自愈动作（服务重启、流量切换、资源扩容），同时保留人工介入兜底。
- **效果闭环**：建立 Bad Case 归集与评测机制，持续优化 Prompt 与工具调用策略。
- **成果**：夜间告警人工介入率从 100% 降至 15%，MTTR 从 45 分钟压缩至 8 分钟。

**应急安全 RAG 知识库 — 多租户检索增强生成系统**

竞品陆续上线了 AI 知识库功能，市场倒逼我们必须补齐这块能力。我主导了 RAG 知识库从 0 到 1 的架构设计与落地，为平台用户提供安全文档的智能检索与问答。核心挑战是在成本约束下，做到安全合规与性能的平衡。我负责整个 RAG 检索链路和多租户数据架构。

- **双轨检索架构**：Qdrant 管 Dense 向量检索（Flat 索引，零参数零维护），ES 管 BM25 关键词检索，应用层 RRF 融合。单租户 200~2000 份文档，Flat 索引查询延迟 1ms，比 HNSW 省调优人力。
- **多租户隔离**：共享 Collection + Payload 过滤（tenant_id），应用层二次校验，新租户 onboarding 从 2 天压缩至 5 分钟。
- **三层缓存降本**：Query 答案缓存（命中率 55%）→ HyDE 缓存（30%）→ LLM 响应缓存（40%），三层叠加 LLM 调用成本降 82%，平均延迟从 800ms 降至 220ms。
- **Bad Case 治理**：新租户冷启动检索质量差（Faithfulness 0.55），通过租户级同义词库 + 冷启动保护期修复至 0.82；异常流量攻击导致 LLM 费用暴涨 300%，通过租户级 QPS 限流 + 缓存 + Token 配额三层防御解决。
- **成果**：120 个活跃租户（含 4 家世界 500 强），端到端延迟 P99 720ms，Recall@10 97%，精确信息准确率 99.8%，单租户月均基础设施成本 25 元。

**自研流程引擎 — 确定性业务流程编排引擎**

平台的业务流程基座，处理访客管理、应急演练、审批流转等确定性工作流。采用 DAG 编排 + 状态机驱动，支持条件分支、异步回调、人工审批节点。

- **分片调度 + 节点执行**：参考 Netty EventLoop 设计分片调度器，按 instance_id 哈希分片，多分片并行，天然保证实例级串行化，无需分布式锁，单机支撑 5000+ TPS。业务方通过 SDK 编写 Handler，引擎自动完成任务派发、回调接收、超时检测。
- **五层租户隔离**：入口限流 → 分片调度 → 分层执行（大客户独立 Lane + 中小客户按套餐分组 + 冷租户 LRU 淘汰）→ 实例配额 → 租户熔断。
- **状态持久化**：主键更新同步落盘 + 乐观锁，节点超时告警 + 人工介入，超时率 < 0.1%。
- **成果**：支撑 200+ 租户、TPS 5000+，单租户流量洪峰 10 倍时其他租户无感知，新产品 MVP 交付周期从 6 周压缩至 2 周，助力签约 2 家世界 500 强客户。
