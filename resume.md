**应急安全 RAG 知识库**

- **项目描述**：基于自研 **CoreAgent 中台服务层**构建的应急安全知识库问答应用。遵循“能复用则复用”原则，不单独维护一套 RAG 入口，而是将 RAG 的检索、精排、生成能力以 Agent Tool 形式注册到 CoreAgent 的 ToolRegistry；用户通过统一的 CoreAgent 入口发起提问，由 AgentExecutor 通过 ReAct 循环编排“检索 → 精排 → 生成”多步流程，最终返回带引用的答案。同一套 CoreAgent 同时支撑运维、运营数据等其他场景。
- **技术应用**：Spring Boot、Spring AI、Qdrant、Elasticsearch、Redis、MySQL、Docker
- **第三方接入**：本地大模型（Qwen）部署、LLM API
- **Web 管理端功能**：知识库管理、文档上传与解析、检索配置、问答会话管理、引用溯源、成本看板
- **核心职责**
  - **统一入口与工具复用**：CoreAgent 作为统一 Agent 入口，RAG 的 Retriever、Reranker、Writer 以 Tool 形式注册到 ToolRegistry，与运维问题分析、运营数据查询等场景共享中台能力
  - **ReAct 多步推理编排**：AgentExecutor 执行 Thought → Action → Observation 循环，编排“检索 → 精排 → 生成”流程，根据中间结果动态决策下一步
  - **RAG 检索召回链路**：完成文档解析与向量化、Qdrant + ES 混合检索、HyDE 查询重写、Query / HyDE / LLM 三级缓存、租户级同义词库与冷启动保护
  - **安全与治理能力**：复用 CoreAgent 的 GuardRail 风险分级、TenantCtrl 配额限流、AgentTracer 全链路追踪、ContextManager 上下文裁剪
- **本地化部署**：基于 Qwen 本地部署，数据不出域，满足 TOB 安全合规要求
- **成果**：端到端延迟 **P99 720ms**、Recall@10 **97%**、精确信息准确率 **99.8%**；LLM 调用成本降 **82%**，平均延迟从 800ms 降至 220ms；单租户月均基础设施成本 **25 元**；新租户冷启动 Faithfulness 从 0.55 提升至 0.82
